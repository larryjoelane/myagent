Hebbian Memory

Memory that wires itself

MyAgent remembers like a brain: memories retrieved together wire together,
and recall is weighted by recency and frequency.

The idea

Most assistants treat memory as a flat list — every note is equal, recall is pure keyword or vector match. MyAgent's memory is a living graph. Each saved chat turn is a neuron. When two memories are recalled together, a synapse grows between them. The more a memory is used, the more energy it carries, and the higher it ranks next time.

Neurons

Every Q&A turn you have with a worker becomes a memory node, embedded for hybrid keyword + semantic search.

Synapses

“Neurons that fire together wire together.” Memories retrieved in the same query gain a Hebbian edge — so asking one can surface the other later.

Energy

Recency × frequency. Memories you return to stay hot and rank higher; ones you ignore cool down — but are never deleted, only re-ranked.

Spreading activation

A match cascades a fraction of its score to wired neighbours — associative recall that a flat vector search would miss. Adjustable in-app.

Why it matters

Auto-context injection uses this graph. Before each prompt is sent, MyAgent retrieves the most relevant past memories — ranked by their synapse score and energy — and prepends them, so the model carries forward what you and it have already worked out. Loosely-related context surfaces through spreading activation; both the match threshold and the spread strength are sliders in the settings drawer.

MyAgent — a local-first coding-agent desktop app with hosted model backends (OpenRouter & Ollama Cloud), hybrid memory search, and a built-in file explorer & editor. Hebbian memory is one of its features.
